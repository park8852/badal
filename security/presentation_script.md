# 모의해킹 프로젝트 중간발표 대본 (초안 v1)

## 0. 인사 및 개요
> 안녕하세요, 모의해킹 프로젝트 **남탕 팀**의 ○○○입니다.  
> 오늘은 “**생성형 AI를 활용한 모의해킹**” 프로젝트의 중간 진행 상황을 공유드리겠습니다.  
> 발표 순서는 프로젝트 개요, 팀 소개, 서비스 소개, 개발 현황, 취약점 진단, 그리고 시나리오 기반 모의해킹 순으로 진행하겠습니다.

## 1. 프로젝트 개요
### 목적
> 이번 프로젝트의 목적은 배달 주문 플랫폼을 직접 구축하고, 취약점을 분석하며, 실제 공격 시나리오를 수행해 **개발-보안-모의해킹 역량을 동시에 강화**하는 것입니다.

### 목표
> 목표는 세 가지입니다.  
> 1) **바로배달** 웹·모바일 서비스를 직접 구축하고  
> 2) 구축한 서비스를 대상으로 취약점을 분석하며  
> 3) 결과를 바탕으로 공격 시나리오를 설계·실행해 모의해킹을 수행하는 것입니다.  
> 한 줄 요약은 “**서비스 구축부터 공격·방어 검증까지 직접 수행해 운영 관점의 취약점을 식별하고 개선하는 실무형 프로젝트**”입니다.

## 2. 팀 소개 & 협업 환경
> 팀명은 **남탕**이고, 백엔드·프론트·모바일·보안 등 역할을 나눠 구성된 **7명**이 함께하고 있습니다.  
> 협업은 GitHub로 코드 공유, Discord로 화면 공유 및 음성 회의, 카카오톡으로 일일 이슈를 소통하고 있습니다.

## 3. 서비스 소개 & 선정 이유
### 서비스 개요
> ‘바로배달’은 **배달 주문부터 가맹점 관리, 정산까지** 전 과정을 지원하는 종합 배달 플랫폼입니다.  
> 사용자용 **Android 앱**, 사장님·관리자용 **웹 대시보드**로 구성되어 있습니다.

### 선정 이유
> 선택한 이유는 세 가지입니다.  
> 1) 배달 서비스는 실제로 널리 사용되는 **대중적 서비스**이고  
> 2) **개인정보와 결제 정보 등 민감 데이터**를 다루며  
> 3) 해킹 피해 시 파급력이 매우 크기 때문입니다.  
> 참고로 2024년 개인정보보호위원회 보도자료에서도 **배달·O2O 서비스 관련 침해 신고가 증가** 추세라고 언급되어 있고, 이에 따라 실제 운영 환경을 가정한 보안 검증이 필요하다고 판단했습니다.

## 4. 개발 진행 현황
### 기술 스택 & 아키텍처
> - 사용자 앱: **Android Kotlin**  
> - 사장님/관리자 웹: **Typescript + Next.js**  
> - 백엔드 API: **Spring Boot + Java**, REST 구조  
> - 데이터베이스: **MariaDB**  
> - 인프라: AWS 기반으로 구성했습니다.

### 산출물 (슬라이드 이미지)
> - **클라우드 구성도**: API 서버, DB, 스토리지, 모의해킹 도구가 어떻게 연결되는지 보여줍니다.  
> - **유스케이스 다이어그램**: 회원·사장님·관리자 역할별 주요 기능을 도식화했습니다.  
> - **시스템 흐름도**: 주문 생성 → 결제 → 정산까지의 프로세스를 표현했습니다.

## 5. 취약점 진단
### 모바일 (Burp Suite + Frida)
> - Burp Suite: API 요청 intercept & 변조  
> - Frida: 런타임 후킹, 민감 데이터 흐름 추적  
> - 주요 결과  
>   - JWT 평문 저장 (DataStore)  
>   - 로그인 요청 로그에 자격증명 노출  
>   - 루팅·Frida 탐지 부재

### 웹/백엔드
> - `OrderController` 미인증 호출 가능  
> - 로그아웃 이후에도 토큰 재사용 가능  
> - 관리자 인가 로직 누락  
> - 비밀번호 정책 부재
