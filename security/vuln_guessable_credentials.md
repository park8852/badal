# 유추가능한 인증정보 이용 점검 결과

## 점검 항목
- 유추가능한 인증정보 이용

## 진단 결과
- 취약

## 점검 목적
- 비밀번호·PIN 생성 시 복잡성 정책이 적용되어 추측 가능한 값 사용을 제한하는지 확인함

## 보안 위협
- 무차별 대입, 사전 공격, 소셜 엔지니어링으로 계정 탈취가 발생할 수 있음

## 판단 기준
- **양호**: 최소 길이, 문자 조합, 반복·연속 제한 등 정책을 만족해야 하며 서버에서 검증함
- **취약**: 아이디와 동일한 문자열, `1234`, 생일 등 단순 패턴을 그대로 허용함

## 조치 방법
- 서버에서 비밀번호 정책을 강제하고 최근 N개 비밀번호 재사용을 막을 것  
- 앱은 약한 비밀번호 입력 시 경고를 제공하고 정책을 안내할 것

## 진단 세부 사항
- 회원가입 API(`POST /api/member/register`)와 모바일 가입 화면 모두 비밀번호 복잡성 검증을 수행하지 않음  
- 서버 `MemberService.register`는 아이디 중복만 확인하고 비밀번호는 그대로 저장함  
- 모바일 앱도 입력값을 검증 없이 전송함  
- 길이 1~2자의 숫자, 아이디와 동일한 문자열 등 쉽게 추측 가능한 비밀번호로 가입이 가능함

### 재현 절차
1. 회원가입 요청(`POST /api/member/register`)의 `userpw` 값을 `1234`, `aaaa`, `userid`와 동일한 값 등으로 변경함  
   ![Step1](./evidence/guessable_creds_step1_request.png "약한 비밀번호로 회원가입 요청")
2. 서버에서 200 응답과 함께 계정이 정상 생성되는 것을 확인함  
   ![Step2](./evidence/guessable_creds_step2_response.png "약한 비밀번호로 회원가입 성공")

## 영향
- 공격자는 사전 공격이나 무차별 대입(brute force)으로 쉽게 계정을 탈취할 수 있음  
- 사용자가 동일 패턴의 비밀번호를 재사용하면 다른 서비스 계정까지 위험에 노출됨

## 개선 권고
- 서버 측에서 비밀번호 최소 길이, 대·소문자·숫자·특수문자 조합, 반복·연속 제한 등을 강제할 것  
- 비밀번호 변경 시 최근 N개 비밀번호 재사용을 금지하고, 실패 횟수 제한·계정 잠금 정책을 도입할 것  
- 모바일 앱에서도 약한 비밀번호 입력 시 경고 메시지와 정책 안내를 제공할 것
