# 메모리 내 중요정보 노출 여부 점검 결과

## 점검 항목
- 메모리 내 중요정보 노출 여부

## 진단 결과
- 취약

## 점검 목적
- 앱 실행 중 디버그 로그, 메모리 덤프에서 민감 정보가 노출되지 않는지 확인함

## 보안 위협
- 악성 앱이나 공격자가 메모리·로그를 훔쳐 토큰, 비밀번호 등을 획득할 수 있음

## 판단 기준
- **양호**: 로그에 민감 값이 남지 않고, 메모리 덤프에서도 평문 노출이 없음
- **취약**: 로그에서 토큰이 출력되거나, 메모리 덤프로 평문 비밀번호를 찾을 수 있음

## 조치 방법
- 민감 데이터를 사용한 후 즉시 초기화하고 로그 출력 금지 정책을 적용할 것  
- 릴리스 빌드에서 `debuggable=false`, ProGuard/난독화, `FLAG_SECURE` 등을 적용할 것

## 진단 세부 사항
- 앱이 로그인·회원가입 과정에서 사용자 자격증명을 그대로 로그에 출력함  
- 로그는 메모리에 기록된 뒤 logcat·원격 로거로 전송될 수 있어, 다른 앱이나 루팅된 환경에서 손쉽게 열람됨

### 재현 절차
1. 에뮬레이터/기기를 연결하고 아래 명령으로 로그 모니터링을 시작함  
   ```bash
   adb logcat | findstr /i "login"
   ```  
   ![Step1](./evidence/memory_leak_step1_logcat.png "adb logcat 실행 화면")
2. 앱에서 로그인 또는 회원가입을 진행함  
   ![Step2](./evidence/memory_leak_step2_login.png "앱에서 로그인 시도")
3. 콘솔에 `Login Request: LoginRequest(userid=aaaa, userpw=aaaa)` 등 민감 값이 그대로 출력되는 것을 확인함  
   ![Step3](./evidence/memory_leak_step3_log.png "로그에 토큰/비밀번호가 노출된 화면")

## 영향
- 로그를 열람할 수 있는 내부 직원·외부 공격자(루팅, 디버깅, 로그 전송 서버 등)가 인증 정보를 획득해 계정을 탈취할 수 있음

## 개선 권고
- 로그인·회원가입·토큰 갱신 등 민감 로직에서 로그 출력 제거 및 `BuildConfig.DEBUG` 조건 검토를 수행할 것  
- 민감 데이터는 사용 후 즉시 메모리에서 지우거나 마스킹 처리하고, 디버그 빌드와 릴리스 빌드의 로깅 정책을 분리할 것  
- 릴리스 빌드에서 `debuggable=false`, ProGuard 난독화 적용, `FLAG_SECURE` 설정으로 스크린샷·메모리 덤프 보호를 강화할 것
