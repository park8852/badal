# OS 변조 탐지 기능 점검 결과

## 점검 항목
- OS 변조 탐지 기능 적용 여부

## 진단 결과
- 취약

## 점검 목적
- 앱이 실행 중 루팅·탈옥, 디버깅 도구, 악성 프레임워크 등 비정상 환경을 감지하고 적절히 대응하는지 확인함

## 보안 위협
- 루팅된 기기에서는 파일 접근, 후킹, 메모리 조작이 용이해 인증 정보 유출, 거래 변조, 악성 코드 삽입 등이 발생할 수 있음

## 판단 기준
- **양호**: 루팅 탐지 라이브러리를 통해 위험 신호를 발견하면 앱 실행을 차단하거나 최소 기능만 제공하며 사용자에게 경고함
- **취약**: 루팅, 디버거, Frida 등 도구가 활성화되어도 앱이 아무 조치 없이 정상 동작함

## 조치 방법
- 탐지 시 실행을 중단하거나 제한 모드로 전환하고, 감지 이벤트를 서버에 보고하여 이상 거래 모니터링과 연계할 것

## 진단 세부 사항
- 루팅된 환경에서 앱이 정상 동작하며 Frida 후킹 도구를 통해 로그인 자격 정보 등 민감 데이터가 노출됨
- 앱에는 루팅·후킹 탐지 로직이 전혀 없고 실행 차단도 이루어지지 않음

### 재현 절차
1. Nox 에뮬레이터에서 루트 모드를 활성화함  
   ![Step1](./evidence/os_tamper_step1_root.png "루트 모드 활성화 화면")
2. 루트 상태의 기기에서 Frida 서버를 실행하고 PC에서 앱에 후킹 스크립트를 주입함  
   ![Step2](./evidence/os_tamper_step2_frida.png "Frida 서버 실행 화면")
3. 앱에서 로그인하면 Frida 콘솔에 `userid=aaaa`, `userpw=aaaa` 등 아이디·비밀번호가 그대로 출력되어 후킹이 성공함  
   ![Step3](./evidence/os_tamper_step3_capture.png "Frida 콘솔 출력")

## 영향
- 공격자는 루팅된 환경에서 후킹으로 인증 토큰·계정 정보를 탈취하고 거래를 변조할 수 있음
- 악성 프레임워크를 주입해 앱 로직을 훼손하거나 백도어를 설치할 위험이 있음

## 개선 권고
- 루팅·탈옥·디버깅·에뮬레이터 탐지 라이브러리를 도입하고, 감지 시 앱 실행을 중단하거나 제한 모드로 전환할 것  
- 탐지 이벤트를 서버에 즉시 보고하여 이상 거래 모니터링 시스템과 연계할 것  
- 인증 정보는 최소한으로 저장하고, 키체인·하드웨어 보안 모듈 등을 활용해 보호할 것
